Schul-KI Portal — Anleitung

Für wen ist diese Anleitung? Für alle, die das Schul-KI Portal zum ersten Mal einrichten oder nutzen möchten — auch ohne technisches Vorwissen.

Was ist das Schul-KI Portal?

Das Schul-KI Portal ist eine App, die direkt im Browser läuft. Sie verbindet sich mit einer lokalen KI (Open WebUI + LM Studio), die auf dem Schulserver oder einem eigenen Computer läuft. Alle Daten bleiben in der Schule — nichts wird in die Cloud gesendet.

Die App hilft beim Schreiben von Elternbriefen, Zeugnistexten, Förderplanen und vielem mehr.

Bevor du startest: Was muss laufen?

Damit die App funktioniert, müssen zwei Programme auf dem Computer gestartet sein:

Programm Wozu Muss laufen?
LM Studio Das KI-Gehirn ✅ Ja, Server muss gestartet sein
Docker Desktop Läuft Open WebUI ✅ Ja, grünes Symbol oben rechts
Open WebUI Startet automatisch mit Docker ✅ Automatisch
Qwen3-TTS Lokale Text-zu-Sprache-Engine (Port 8000) Optional — Browser-Stimme als Fallback
Obsidian Wissensarchiv (optional) Optional

Wenn du dir nicht sicher bist, ob alles läuft: Öffne den Browser und gib http://localhost:3000 ein. Wenn du eine Chat-Oberfläche siehst, ist alles bereit.

Tipp: Eine detaillierte Installationsanleitung für LM Studio, Docker und Open WebUI findest du in der separaten Setup-Anleitung.

Die App öffnen

Öffne die Datei schul-ki-portal.html per Doppelklick im Browser — oder rufe die Adresse auf, unter der die Datei auf eurem Schulserver liegt.

Du siehst dann den Anmeldebildschirm.

Schritt 1 — Ersteinrichtung (einmalig)

Beim allerersten Start musst du ein paar Einstellungen vornehmen. Das ist nur einmal nötig.

Klicke unten auf dem Anmeldebildschirm auf „Einstellungen / Ersteinrichtung".

Tab „SSO / Anmeldung"

Hier richtest du ein, mit welchem Schul-Konto man sich anmelden kann.

Feld Was du einträgst
OIDC Issuer URL Die Adresse eures Anmeldesystems (von der IT-Abteilung)
Client-ID Eine Nummer aus der App-Registrierung (von der IT-Abteilung)
Redirect-URI Wird automatisch ausgefüllt — einfach so lassen
Schulname z. B. „Mustergymnasium Düsseldorf"

Noch kein SSO? Kein Problem — du kannst die App auch ohne Anmeldung im Demo-Modus starten (siehe unten).

Tab „Open WebUI"

Feld Was du einträgst
Open WebUI URL http://localhost:3000 (Standard, einfach so lassen)
API-Key In Open WebUI unter: Profilbild → Einstellungen → API-Schlüssel
Standard-Modell z. B. meta-llama-3.1-8b-instruct
TTS-URL http://localhost:8000 (nur wenn Qwen3-TTS installiert ist)

Klicke auf Speichern. Fertig!

Schritt 2 — Anmelden

Du hast zwei Möglichkeiten:

Mit SSO anmelden — Klicke auf den blauen Button. Du wirst zur Schulanmeldung weitergeleitet und danach automatisch zurückgebracht.

Demo-Modus — Klicke auf „Demo-Modus (ohne SSO starten)". Du bist sofort drin, ohne Passwort.

Use Cases nutzen — so funktioniert es

Jeder Use Case folgt demselben Prinzip:

1. Felder ausfüllen — Gib die nötigen Informationen ein, zum Beispiel Klasse, Anlass, Ton.

2. Modell wählen — Unten links kannst du das KI-Modell auswählen. Das Standardmodell passt für die meisten Aufgaben.

3. Generieren klicken — Die KI erstellt in wenigen Sekunden einen Entwurf.

4. Prüfen und kopieren — Lies das Ergebnis durch, passe es bei Bedarf an und klicke auf „Kopieren". Dann kannst du es in Word, LibreOffice oder deine E-Mail einfügen.

Wichtig: Die KI liefert immer einen Entwurf. Vor dem Versenden immer selbst prüfen, ob alles stimmt — besonders bei Namen, Daten und Noten.

Beispiel: Elternbrief schreiben

  1. Klicke links auf „1 Elternbriefe"
  2. Trage ein: Klasse 8b, Anlass Klassenfahrt nach Berlin vom 14.–17. Mai, Kosten 185 €, Rückmeldung bis 2. Mai
  3. Wähle Ton: freundlich, informativ, professionell
  4. Wähle Format: Formeller Brief mit Briefkopf
  5. Klicke „▶ Generieren"
  6. Ergebnis prüfen → „Kopieren" → in Word einfügen

Chat nutzen

Der Chat funktioniert wie ChatGPT — du kannst einfach drauflos schreiben.

Direktnachricht schreiben

Klicke auf „💬 Chat" im Menü, tippe deine Frage ins Textfeld und drücke Enter.

Skills im Chat aktivieren

Skills sind vorgefertigte Prompt-Bausteine, die du im Chat aktivieren kannst.

So geht es:

  • Tippe / ins Textfeld — ein Menü mit deinen gespeicherten Skills erscheint
  • Tippe weiter um zu suchen, z. B. /Elternbrief
  • Klicke den gewünschten Skill an — er wird als gelber Chip angezeigt
  • Schreibe deine Nachricht dazu und drücke Enter

Du kannst auch mehrere Skills gleichzeitig aktivieren. Der Skill-Kontext wird automatisch mitgeschickt.

Alternativ: Klicke auf „✦ Skill auswählen" unter dem Textfeld.

Skills-Bibliothek

Hier speicherst du deine eigenen Prompt-Bausteine für häufig wiederkehrende Aufgaben.

Neuen Skill anlegen

  1. Klicke auf „+ Neuer Skill"
  2. Trage ein:
  3. Titel z. B. „Formeller Briefabschluss"
  4. Kategorie z. B. „Formulierungen"
  5. Tags z. B. „brief, eltern, formal"
  6. Inhalt — der eigentliche Prompt-Text
  7. Verwendungshinweis z. B. „Für UC1 und UC6 geeignet"
  8. Klicke Speichern

Platzhalter nutzen

Im Skill-Inhalt kannst du Platzhalter wie {{Klasse}} oder {{Name}} verwenden. Wenn du den Skill einfügst, ersetzt du die Platzhalter manuell mit dem konkreten Wert.

Skill in einen Use Case einfügen

Öffne einen Use Case, scrolle zu einem Textfeld und klicke auf „↗ Einfügen" beim gewünschten Skill.

Skills aus der Online-Bibliothek

Unter „Skills-Bibliothek ↗" findest du fertige Skills von anderen Lehrkräften, die du herunterladen und importieren kannst.

Skills exportieren/importieren: Klicke auf „↓ Export" bzw. „↑ Import" — das erzeugt eine .json-Datei, die du mit Kolleginnen und Kollegen teilen kannst.

🔊 Text → Sprache (TTS)

Die App kann KI-Ergebnisse und eigene Texte laut vorlesen — zum Beispiel um Elternbriefe abzuhören oder Schülerinnen und Schüler beim Zuhören zu unterstützen.

Wie es funktioniert

Die App nutzt automatisch die beste verfügbare Stimme:

Engine Voraussetzung Qualität
Qwen3-TTS (primär) Server läuft lokal auf Port 8000 ⭐⭐⭐ Natürlich, multilingual DE/EN
Browser-Stimme (Fallback) Immer verfügbar, kein Setup nötig ⭐⭐ Systemstimme

Tipp: Läuft kein Qwen3-TTS-Server, schaltet die App automatisch auf die Browser-Stimme um — du musst nichts einstellen.

Verfügbare Stimmen (Qwen3-TTS)

Alle Stimmen unterstützen Deutsch und Englisch: Chelsie, Ryan, Aiden, Ethan, Serena. Du kannst die Standardstimme in den Einstellungen festlegen.

Text vorlesen lassen

Im Chat: Jede KI-Antwort hat automatisch einen „🔊 Vorlesen"-Button. Klicke ihn, um die Antwort direkt anzuhören. Ein zweiter Klick stoppt die Wiedergabe.

Im TTS-Panel:

  1. Klicke links auf „🔊 Text → Sprache"
  2. Tippe oder füge deinen Text ins Textfeld ein — oder klicke auf „Aus Use-Case einfügen", um das letzte KI-Ergebnis direkt zu übernehmen
  3. Wähle Stimme und Geschwindigkeit
  4. Klicke „▶ Vorlesen"

Einrichtung Qwen3-TTS (einmalig, 2 Befehle)

Öffne ein Terminal und führe nacheinander aus:

pip install qwen-tts
qwen-tts-demo Qwen/Qwen3-TTS-12Hz-0.6B-CustomVoice --port 8000

Danach in der App: Einstellungen → Tab „Open WebUI" → Feld TTS-URL auf http://localhost:8000 setzen und speichern.

Hinweis: Den TTS-Server musst du bei jedem Neustart des Computers erneut starten (zweiter Befehl). Der erste Befehl ist nur einmal nötig.

🎤 Sprache → Text (STT)

Diktiere direkt im Chat oder im Medien-Panel — die App überträgt deine Sprache in Text und leitet ihn optional direkt an einen Use Case weiter.

Diktat im Chat

  1. Klicke auf das Mikrofon-Symbol im Chat-Eingabefeld
  2. Das Symbol pulsiert rot — das Mikrofon ist aktiv
  3. Sprich deine Nachricht
  4. Klicke erneut auf das Symbol (oder pausiere kurz) — die Transkription erscheint automatisch im Textfeld
  5. Prüfe den Text und drücke Enter zum Senden

Transkription im STT-Panel

Für längere Aufnahmen (z. B. Konferenzprotokolle) gibt es das eigenständige STT-Panel:

  1. Klicke links auf „🎤 Sprache → Text"
  2. Klicke „⏺ Aufnahme starten" — ein Timer zeigt die Aufnahmedauer
  3. Klicke „⏹ Stopp" — Whisper transkribiert die Aufnahme
  4. Prüfe die Transkription im Textfeld
  5. Klicke „→ In Use Case übertragen" und wähle das Zielfeld, z. B. UC3 (Konferenzprotokolle), UC1 (Elternbriefe) oder UC7 (Handschrift/OCR)

Technik im Hintergrund: Die Transkription läuft über Open WebUI's Whisper-Schnittstelle (/audio/transcriptions) — alles bleibt lokal.

🖼 Bildgenerierung

Erstelle Unterrichtsbilder, Infografiken oder Illustrationen direkt in der App — ohne Drittanbieter, alles auf dem Schulserver.

Bild erstellen

  1. Klicke links auf „🖼 Bildgenerierung"
  2. Beschreibe das gewünschte Bild auf Deutsch oder Englisch im Prompt-Feld
  3. Optional: Klicke auf „✨ Prompt mit KI verbessern" — das Sprachmodell optimiert deinen Text für bessere Ergebnisse
  4. Wähle einen Stil aus den Voreinstellungen:
Stil Geeignet für
Aquarell Kunstunterricht, Illustrationen, Geschichten
Fotorealistisch Sachunterricht, Naturwissenschaften
Vektor Diagramme, Icons, Präsentationsfolien
Skizze Erklärzeichnungen, Tafelbilder
Infografik Statistiken, Übersichten, Lernplakate
  1. Klicke „▶ Generieren"
  2. Unter dem fertigen Bild erscheinen zwei Buttons: „↓ Download" speichert das Bild als PNG, „↻ Variation" erzeugt eine neue Version desselben Motivs

Technik im Hintergrund: Die App verbindet sich mit Open WebUI's Bildgenerierungs-API. Unterstützte Modelle: FLUX, Stable Diffusion, Qwen-Image — je nachdem, was auf eurem Server installiert ist.

Wenn ihr Obsidian als Wissensarchiv nutzt, könnt ihr Notizen direkt in der App lesen und KI-Ergebnisse dort speichern.

Voraussetzung: Plugin installieren

In Obsidian: 1. Einstellungen → Community Plugins → Durchsuchen 2. Suche nach „Local REST API" 3. Installieren und aktivieren 4. API-Key kopieren (steht in den Plugin-Einstellungen) 5. Den Key im Schul-KI Portal unter Einstellungen → Open WebUI eintragen

Notiz öffnen und in Prompt einfügen

  1. Klicke auf „◈ Obsidian Vault"
  2. Klicke auf „⟳ Verbinden" — der Dateibaum erscheint
  3. Klicke auf eine Datei — der Inhalt wird angezeigt
  4. Klicke auf „→ In Prompt einfügen" — wähle das Zielfeld

KI-Ergebnis in den Vault speichern

  1. Führe einen Use Case aus (z. B. Protokoll generieren)
  2. Wechsle zu „Obsidian Vault"
  3. Trage unten einen Pfad ein, z. B. Konferenzprotokolle/2026-04-22.md
  4. Wähle das Use-Case-Ergebnis aus
  5. Klicke „↑ In Vault speichern"

Häufige Probleme

„Die KI antwortet nicht"

  • Ist LM Studio geöffnet? Ist der Server gestartet (grüner Button)?
  • Ist Docker Desktop geöffnet (grünes Symbol)?
  • Öffne http://localhost:3000 im Browser — lädt die Seite?

„Modelle erscheinen nicht"

  • In den Einstellungen: Open WebUI URL auf http://localhost:3000 prüfen
  • API-Key in Open WebUI neu generieren und eintragen
  • Auf „Modelle neu laden" klicken (Einstellungen → Seite)

„TTS-Stimme klingt wie Roboter"

  • Die App nutzt gerade die Browser-Fallback-Stimme — Qwen3-TTS läuft vermutlich nicht
  • Terminal öffnen und prüfen: qwen-tts-demo Qwen/Qwen3-TTS-12Hz-0.6B-CustomVoice --port 8000 starten
  • TTS-URL in Einstellungen auf http://localhost:8000 prüfen

„Mikrofon funktioniert nicht"

  • Browser fragt beim ersten Klick nach Mikrofon-Erlaubnis — „Erlauben" klicken
  • Kein Mikrofon angeschlossen? Ein USB- oder Headset-Mikrofon reicht aus
  • Open WebUI muss laufen, da die Transkription über dessen Whisper-API läuft

„Bildgenerierung schlägt fehl"

  • Ist ein Bildgenerierungs-Modell (FLUX, SD oder Qwen-Image) in Open WebUI installiert?
  • Open WebUI → Admin-Einstellungen → Bildgenerierung prüfen
  • Bei Bedarf IT-Abteilung kontaktieren (Modell muss einmalig heruntergeladen werden)

„Obsidian Vault lädt nicht"

  • Ist das Plugin „Local REST API" in Obsidian aktiv?
  • Ist Obsidian geöffnet?
  • API-Key korrekt eingetragen?

„SSO funktioniert nicht"

  • Kein Problem — einfach Demo-Modus nutzen, bis die IT-Abteilung SSO eingerichtet hat.

Datenschutz auf einen Blick

  • Alle Anfragen laufen über den lokalen Schulserver
  • Keine Daten verlassen das Schulnetzwerk
  • Schülerdaten (Noten, Namen) dürfen eingegeben werden, da die KI lokal läuft
  • Vor Konferenzaufnahmen: Einwilligung aller Anwesenden einholen (DSGVO Art. 6)
