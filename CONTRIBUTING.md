# Contributing

Danke für dein Interesse an diesem Projekt.

## Bevor du startest

Bitte beachte:

- keine sensiblen Schul-, Personen- oder Zugangsdaten committen
- keine echten API-Keys, Tokens oder OIDC-Konfigurationswerte ins Repository aufnehmen
- Änderungen an Datenschutz- oder Sicherheitsbehauptungen nur vornehmen, wenn sie technisch und organisatorisch belastbar sind

## Empfohlener Ablauf

1. Repository forken oder Branch erstellen
2. Änderung klar und klein halten
3. Oberfläche lokal testen
4. Dokumentation bei Bedarf mit aktualisieren
5. Pull Request mit nachvollziehbarer Beschreibung eröffnen

## Coding-Hinweise

- Bevorzuge klare, lesbare Änderungen statt großer Umbauten in einem Schritt.
- Wenn du Funktionen ergänzt, dokumentiere sie auch im README.
- Wenn du neue externe Abhängigkeiten einführst, begründe das im Pull Request.

## Themen für Beiträge

- Fehlerbehebungen
- UI-/UX-Verbesserungen
- Dokumentation
- Modularisierung des Codes
- bessere Konfigurationsoptionen
- Accessibility-Verbesserungen

## Bitte vermeiden

- Commits mit Zugangsdaten
- nicht dokumentierte Architekturbrüche
- rechtliche oder datenschutzbezogene Aussagen ohne Prüfung
