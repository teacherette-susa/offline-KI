Schritt 1 — LM Studio installieren

Schritt 1
LM Studio herunterladen und installieren
  1. Öffne im Browser: lmstudio.ai
  2. Klicke auf „Download for Mac" und lade die .dmg-Datei herunter.
  3. Öffne die .dmg-Datei und ziehe LM Studio in den Programme-Ordner.
  4. Starte LM Studio.
Schritt 2
Ein Sprachmodell herunterladen
  1. Klicke links auf das Lupen-Symbol (Discover / Suchen).
  2. Suche nach einem dieser Modelle:
ModellGrößeEmpfehlung
meta-llama-3.1-8b-instruct~5 GBSehr gut für Texte und Verwaltung
mistral-7b-instruct~4 GBSchnell, gut für kurze Aufgaben
deepseek-r1-8b~5 GBGut für komplexere Anfragen
  1. Klicke beim gewünschten Modell auf den Download-Pfeil. Der Download dauert je nach Internetgeschwindigkeit 10–30 Minuten.
Schritt 3
Den lokalen Server starten
  1. Klicke auf das Server-Symbol (Local Server / Entwickler).
  2. Wähle im Dropdown ein Modell aus → „Load Model".
  3. Klicke auf den grünen Button „Start Server".
  4. Zur Kontrolle — öffne im Browser:
http://127.0.0.1:1234/v1/models

Du solltest eine Liste mit deinen Modellen sehen — dann läuft alles korrekt.

⚠️ LM Studio muss immer geöffnet und der Server gestartet sein, wenn Open WebUI genutzt werden soll. Am besten beim Schulstart automatisch starten lassen.

Schritt 2 — Docker Desktop installieren

Docker ist nötig, damit Open WebUI auf deinem Mac läuft. Die Installation ist einmalig und danach unsichtbar im Hintergrund.

  1. Öffne im Browser: docker.com/products/docker-desktop
  2. Klicke auf „Download for Mac" — wähle Apple Silicon oder Intel je nach Mac-Typ.
  3. Öffne die .dmg-Datei und ziehe Docker in den Programme-Ordner.
  4. Starte Docker Desktop.
  5. Warte, bis das Docker-Symbol in der Menüleiste oben rechts grün wird.

ℹ️ Apple Silicon oder Intel? Klicke oben links auf das Apple-Symbol → „Über diesen Mac". Dort steht entweder „Apple M1/M2/M3" (= Apple Silicon) oder „Intel Core" (= Intel).

Schritt 3 — Open WebUI installieren

Open WebUI ist die Chat-Oberfläche, die Lehrkräfte im Browser nutzen. Sie wird mit einem einzigen Befehl über Docker installiert.

Schritt 1
Terminal öffnen

Drücke gleichzeitig: Command ⌘ + Leertaste → tippe „Terminal" → Enter.

Schritt 2
Open WebUI starten

Kopiere diesen Befehl vollständig und füge ihn im Terminal ein, dann Enter:

docker run -d -p 3000:8080 \
  --add-host=host.docker.internal:host-gateway \
  ghcr.io/open-webui/open-webui:main

Beim ersten Mal wird Open WebUI heruntergeladen — das dauert einige Minuten. Danach startet es automatisch.

Schritt 3
Im Browser öffnen
http://localhost:3000

Beim ersten Aufruf kannst du ein Admin-Konto anlegen (Name + E-Mail + Passwort). Dieses Konto bleibt lokal auf deinem Computer.

💡 Beim nächsten Neustart des Macs muss Docker Desktop geöffnet werden, damit Open WebUI wieder läuft. Der docker run-Befehl muss nicht nochmal ausgeführt werden — Docker startet den Container automatisch.

Schritt 4 — Open WebUI mit LM Studio verbinden

  1. Öffne Open WebUI: http://localhost:3000
  2. Klicke oben rechts auf dein Profilbild → Einstellungen → Verbindungen (oder Admin-Bereich → Verbindungen).
  3. Klicke auf „Verbindung hinzufügen".
  4. Trage folgende Werte ein:
FeldWert
VerbindungstypExtern
URLhttp://host.docker.internal:1234/v1
API-Keylmstudio (oder ein beliebiges Wort — darf nicht leer sein)
AnbietertypOpenAI
  1. Klicke auf „Speichern".
  2. Klicke auf die zwei Pfeile (Refresh) neben der URL — nach kurzer Zeit erscheinen die Modelle.

⚠️ Wichtig: Nicht localhost:1234, sondern host.docker.internal:1234 verwenden. Docker kann localhost nicht direkt ansprechen.

✅ Wenn die Modelle in Open WebUI erscheinen, ist alles verbunden. Du kannst jetzt einen ersten Chat-Test machen.

Schritt 5 — Obsidian einrichten (optional)

Obsidian ist das schulinterne Wissenssystem — alle Dokumente werden dort abgelegt und sind per KI durchsuchbar.

Schritt 1
Obsidian installieren
  1. Öffne im Browser: obsidian.md
  2. Lade Obsidian herunter und installiere es wie eine normale App.
  3. Starte Obsidian und lege einen neuen Vault an, z.B. Schule-Wissen.
Schritt 2
Ordnerstruktur anlegen

Erstelle im Vault folgende Unterordner (per Rechtsklick → Neuer Ordner):

Schule-Wissen/
├── Lernenden-Profile/
├── Konferenzprotokolle/
├── Lehrplaene/
├── Dienstanweisungen/
├── Materialarchiv/
└── Vorlagen/
Schritt 3
Community Plugins aktivieren
  1. Zahnrad-Symbol unten links → Community-Plugins
  2. Button „Community-Plugins aktivieren" klicken → Bestätigen.
Schritt 4
Plugin „Local Providers" installieren (Pflicht für Local GPT)
  1. Einstellungen → Community-Plugins → Durchsuchen
  2. Suche: Local Providers → Installieren → Aktivieren
  3. Zahnrad bei Local Providers → URL eintragen:
http://127.0.0.1:1234/v1

Kein API-Key erforderlich — Feld leer lassen.

Schritt 5
Plugin „Local GPT" installieren
  1. Einstellungen → Community-Plugins → Durchsuchen
  2. Suche: Local GPT → Installieren → Aktivieren
  3. Zahnrad bei Local GPT → Provider: Local Providers → Modell wählen → Speichern.

Text in Obsidian markieren → Rechtsklick → Local GPT → Aktion auswählen. Die KI antwortet direkt im Dokument.

💡 Obsidian nutzt direkt localhost, weil es nicht in Docker läuft. Deshalb http://127.0.0.1:1234/v1 und nicht die Docker-Adresse.

Schritt 6 — Qwen3-TTS einrichten (optional · neu)

Qwen3-TTS ist eine lokale Text-zu-Sprache-Engine mit natürlich klingenden Stimmen auf Deutsch und Englisch. Die Einrichtung dauert weniger als 5 Minuten. Läuft kein TTS-Server, fällt das Schul-KI Portal automatisch auf die Browser-Systemstimme zurück.

Schritt 1
Qwen3-TTS installieren (einmalig)

Öffne das Terminal und führe nacheinander aus:

pip install qwen-tts
qwen-tts-demo Qwen/Qwen3-TTS-12Hz-0.6B-CustomVoice --port 8000

💡 Der erste Befehl (pip install) ist nur einmal nötig. Der zweite Befehl startet den TTS-Server und muss nach jedem Neustart des Macs erneut ausgeführt werden.

Schritt 2
TTS-URL im Schul-KI Portal eintragen

Im Schul-KI Portal: Einstellungen → Tab „Open WebUI" → Feld TTS-URL auf http://localhost:8000 setzen → Speichern.

Verfügbare Stimmen

Alle fünf Stimmen unterstützen Deutsch und Englisch:

Die Standardstimme kann im Schul-KI Portal unter Einstellungen festgelegt werden.

Hinweis: Sprache → Text (STT) und Bildgenerierung

ℹ️ STT (Whisper) und Bildgenerierung (FLUX / Stable Diffusion / Qwen-Image) benötigen keine separate Installation — sie laufen vollständig über Open WebUI. Voraussetzung: In Open WebUI müssen die Modelle einmalig unter Admin-Einstellungen → Audio bzw. → Bildgenerierung aktiviert werden. Bei Fragen: IT-Abteilung kontaktieren.

Häufige Probleme und Lösungen

! Open WebUI lädt nicht ▶

Docker Desktop ist nicht gestartet. Docker öffnen und warten bis das Symbol in der Menüleiste grün wird.

! Modelle erscheinen nicht in Open WebUI ▶

URL prüfen: muss http://host.docker.internal:1234/v1 sein (nicht localhost). LM Studio Server muss laufen und ein Modell geladen haben.

i LM Studio antwortet nicht ▶

Browser-Test: http://127.0.0.1:1234/v1/models aufrufen. Wenn leer: in LM Studio Modell neu laden und Server neu starten.

i Obsidian Local GPT findet kein Modell ▶

Sicherstellen dass Local Providers installiert ist (Pflicht!). API-URL gehört in Local Providers, nicht in Local GPT. Einstellungen → Community-Plugins → Local Providers (Zahnrad) → URL: http://127.0.0.1:1234/v1. Kein API-Key nötig.

i TTS klingt wie ein Roboter / kein Qwen3-TTS ▶

Die App nutzt die Browser-Fallback-Stimme — Qwen3-TTS läuft nicht. Terminal öffnen und ausführen:

qwen-tts-demo Qwen/Qwen3-TTS-12Hz-0.6B-CustomVoice --port 8000

Danach im Schul-KI Portal: Einstellungen → TTS-URL auf http://localhost:8000 prüfen.

! Mikrofon / Sprache → Text funktioniert nicht ▶

Browser fragt beim ersten Klick nach Mikrofon-Erlaubnis — „Erlauben" klicken. Open WebUI muss laufen (Whisper-Transkription läuft darüber). Kein Mikrofon erkannt: USB- oder Headset-Mikrofon anschließen und Browser neu laden.

i Bildgenerierung schlägt fehl ▶

In Open WebUI muss ein Bildgenerierungs-Modell aktiviert sein: Admin-Einstellungen → Bildgenerierung. Unterstützte Modelle: FLUX, Stable Diffusion, Qwen-Image. Beim ersten Einsatz muss das Modell einmalig heruntergeladen werden — IT-Abteilung kontaktieren.

i open-webui: command not found ▶

Nicht über pip installieren — Docker-Variante verwenden (Schritt 2 und 3 dieser Anleitung).
